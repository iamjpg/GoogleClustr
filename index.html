Google Clustr

Performant clustering for large datasets on Google Maps.

Features

Performant

By not needing to create a Google Maps Marker instance for every point, the performance for large data sets is much (like, by a lot) better than the official clustering library.

Marker Spiderfy

Once below the cluster threshold the library uses the Marker Spiderfy library to work with overlapping Markers.

Info Window Agnostic

Using the integrated PubSub API you can control hover and click interactions as you please. We're using Tippy, but you can use whatever you want.

Configurable

Branding is important. You have the power to make your clusters and markers look spectacular by adjusting options.

PubSub Events

As mentioned above, there is a built in Publish/Subscribe library you can tap into. Listen to click, hover, marker-show, and count events to add whatever interactive sauce you'd like.

Smart Zoom

When clicking on a cluster, depending on the convex hull of the markers, you may need to zoom into the best view for the polygon, or zoom in one level if members of the polygon are outside the visible map. We got ya covered.
